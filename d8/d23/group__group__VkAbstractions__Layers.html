Detailed Description

Functionality found in this module is related to layers and extensions in Vulkan. This module is part of the core Vulkan abstractions.

One guide to summarise the use of layers can be found in the Vulkan-Guide GitHub repo. This answer on StackExchange suitably describes the use of Vulkan extensions.

See also
KhronosGroup/Vulkan-Guide

◆ oriCheckInstanceExtensionEnabled()

bool oriCheckInstanceExtensionEnabled ( oriState *  state,
const char *  extension 
)

Similar to in oriCheckLayerEnabled(), the extension does not have to be actually 'enabled' yet; the function will still return true if it has at least been specified (with oriFlagInstanceExtensionEnabled()) - this means no instances have to have been created with state for the function to work properly.

Unlike with layers, however, the validation of the specified instance extensions is not checked outside of oriPruneInstanceExtensions() (which is implicitly called in oriCreateInstance()). If you are calling this function before oriCreateInstance(), you should prune the instance extensions first to avoid false positives on invalid extensions. Note, though, that any extensions that are provided by layers not yet specified (even if said layers are specified later) will be removed from the state's list of instance extensions. See oriPruneInstanceExtensions() for more.

Warning
If state is NULL, false will be returned.
Parameters
statethe state object to query.
extensionthe instance extension to search for.
Returns
true if the instance extension has been enabled
false if the instance extension has not been enabled

The layer does not have to be actually 'enabled' yet; the function will still return true if the layer has at least been specified (with oriFlagLayerEnabled()) - this means no instances have to have been created with state for the function to work properly.

◆ oriPruneInstanceExtensions()

bool oriPruneInstanceExtensions ( oriState *  state)

This function validates and removes any unavailable or non-existant instance extensions that have been specified for the oriState object state.

This allows for you to specify instance extensions before the layers that provide them - essentially, Vulkan instance extensions are either provided by the implementation or a layer.

This function is implicitly called in oriCreateInstance() in case you forget.

Warning
If you specify instance extensions that are provided by a layer, and you call this function before specifying said layer, these instance extensions will be removed from the state's internal list, and therefore will not be enabled, even if you specify the layer after this function.
Parameters
statethe state to validate.
Returns
true if any extensions have been removed from the state's list.
false if no extensions have been removed.
