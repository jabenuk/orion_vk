Summary

The original goal of Orion was to run OpenGL without the annoyances of raw OpenGL code, but it wasn't (and still isn't) really intended to be used in production or actual development. It's just a fun project for me to work on in my own time.

Now, I'm shifting my focus towards Vulkan, due to its (somewhat ridiculous, in a fun way) explicit-ness and its priority on performance and optimisation. I also think the Vulkan specification is lightyears ahead of the OpenGL specification - which is pretty abysmal at times - so hopefully development will be a little less painful. Plus, the new Vulkan design is nice for me - I prefer having structs with decent names and organisation than having a bunch of functions (although the OpenGL functions were definitely more simple).

Something else I want to have is actually decent documentation - so we'll see how that goes.

Including

There is one file to be included to make use of the 'core' library: orion.h.

This is the public header for including Orion. All public library functionality is declared here, so the page for this header will have all core functionality documented, pretty much.

Modules

Core library functions have been organised into the following modules:

  • Library management: functions that handle the initialisation, management, and eventual termination of the library.
  • Error handling: functionality to catch errors thrown by Orion at runtime.
  • Vulkan abstractions: abstractions of the Vulkan API.

Debugging

Orion provides the ability to set your own error callback, similar to OpenGL, Vulkan, GLFW, etc. Despite the name, this callback is not just used for errors, but also debug log messages, notifications, and warnings, so it's wise to consider it a 'debug callback' instead of specifically an 'error callback'. (Although it is referred to as an 'error callback' in this documentation).

See the debugging page for more information. You should also see the error index for a comprehensive list of library errors.
