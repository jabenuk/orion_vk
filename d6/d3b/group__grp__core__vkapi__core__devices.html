Detailed Description

Functionality in this module is related to the management and usage of graphics devices with Vulkan, including the enumeration of available and suitable physical devices and choosing the best one for the application.

This module is part of the core Vulkan API module.

◆ oriPhysicalDeviceSuitabilityCheckfun

typedef bool(* oriPhysicalDeviceSuitabilityCheckfun) (const VkPhysicalDevice device)

A function of this signature is to be used to select suitable graphics device/s to be used in the application.

Devices reported to this function will only include those with Vulkan support, so if there are no graphics devices with Vulkan support, this function will not be called.

You should return true if the device supports the extensions you require, the device supports the required queue families, etc. You can make the function as simple or as advanced as you need - just make sure you return true if the device is suitable and false if it isn't.

Parameters
devicethe device to be queried

This function creates a logical device (VkDevice) to connect to a physical device. The resulting logical device will be managed by Orion.

The deviceNext parameter can be used to extend the device structure. For instance, you could use it to add a VkDeviceGroupDeviceCreateInfo, which would allow you to connect the logical device to multiple physical devices.

See Vulkan Docs/VkDeviceCreateInfo and, occasionally, Vulkan Docs/VkDeviceGroupDeviceCreateInfo for more info.

Parameters
deviceOutpointer to the VkDevice into which the resulting device will be returned.
deviceFlagsa bitmask of VkDeviceCreateFlags indicating the behaviour of the logical device.
physicalDevicethe physical device to interface with.
queueCreateInfoCountthe size of the queueCreateInfos array.
queueCreateInfosarray of VkDeviceQueueCreateInfo structures describing the queues to be requested along with the logical device.
extensionCountthe size of the extensionNames array.
extensionNamesarray of the names of device extensions to be enabled for the logical device.
enabledFeaturesNULL or a pointer to a VkPhysicalDeviceFeatures structure containing flags of device features to be enabled.
deviceNextNULL or a pointer to a structure to extend the device creation info structures.
Returns
OK if executed successfully
NULL_POINTER if physicalDevice is NULL or if queueCreateInfos is NULL but queueCreateInfoCount is more than 0, or the extension equivalents
SKIPPED if a device has already been allocated by Orion at deviceOut.
ERROR if the device failed to be created by Vulkan.

◆ oriEnumerateSuitablePhysicalDevices()

const oriReturnStatus_t oriEnumerateSuitablePhysicalDevices ( const VkInstance *  instance,
const oriPhysicalDeviceSuitabilityCheckfun  checkFun,
unsigned int *  countOut,
VkPhysicalDevice **  devicesOut 
)

This function retrieves an array of physical devices accessible to a Vulkan instance that are considered suitable for the application, making use of the checkFun function to determine their suitability.

If you set checkFun to NULL, all available physical devices will be returned.

Note
If there are no suitable (or available if checkFun is NULL) devices, then devicesOut will be set to NULL. countOut will, of course, be set to 0.
Warning
The devicesOut array will be allocated by this function and it is up to you to free it.
Parameters
instancethe VkInstance object to query.
checkFunNULL, or a oriPhysicalDeviceSuitabilityCheckfun function to determine device suitability on your terms.
countOutNULL or a pointer to the variable into which the amount of devices will be returned
devicesOutNULL or a pointer to the array into which the Vulkan physical device objects will be returned
Returns
OK if executed successfully
NULL_POINTER if instance is NULL
NO_OUTPUT if both countOut and devicesOut are NULL
ERROR if there was an unspecified error, or if memory for the device failed to allocate
