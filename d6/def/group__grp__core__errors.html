Detailed Description

Content found in this module is related to catching and debugging errors thrown by the library at runtime. You should also see the Debugging page for more information about general debugging.

It is part of the core Orion library interface.

◆ oriDebugCallbackfun

typedef void(* oriDebugCallbackfun) (const char *name, const unsigned int code, const char *message, const oriSeverityBit_t severity, void *pointer)

This function pointer defines a callback function for general runtime errors.

It function signature must be followed when setting a custom Orion debug callback, which can be done in oriSetDebugCallback().

A list of error severities (passed to the severity parameter) can be seen in the Debugging/Severities section.

Parameters
namethe name of the error ID, if applicable*
codethe error code, if applicable*
messagea message for debugging
severitythe severity of the message
pointerNULL or a user-specified pointer (can be specified in oriSetDebugCallback())

*only applicable when severity is ERROR or FATAL. Messages with severities of WARNING or below are not standardised, and do not have a set name or code.

◆ oriReturnStatus_t

enum oriReturnStatus_t

All Orion functions (that could result in errors) return an oriReturnStatus_t enum. If the function returns any value other than ORION_RETURN_STATUS_OK (0), something has gone wrong. The name of the returned enum will give some information, but this is generally vague and you should check the debug output for more information.

This can be used to handle errors in your program.

To see a list of all possible return statuses, see the Error index.

◆ oriConfigureDebugMessages()

const oriReturnStatus_t oriConfigureDebugMessages ( const oriSeverityBit_t  severities)

This function enables Orion debug output with the specified criteria.

A list of error severities can be seen in the Debugging/Severities section.

The debug callback specified with oriSetDebugCallback() (or the default one if this was not done) will be called when a debug message that matches the specified criteria is enqueued by the Orion library.

You may call this function multiple times throughout runtime - the severities that will be shown will be overriden each time the function is executed.

Parameters
severitiesa bit field of oriSeverityBit_t enumerators.
Returns
OK if executed successfully

void *  pointer

This function sets the global Orion debug callback function.

User data can be passed to the callback through the pointer parameter. This will be recieved in the callback with the parameter of the same name.

Pass NULL to the callback parameter to use the default, built-in debug callback.

Pass NULL to the pointer parameter if you do not want to pass any data to the callback.

Parameters
callbackNULL or the callback function to use, definition specified as oriDebugCallbackfun.
pointerNULL or specified user data that will be sent to the callback.
Returns
OK if executed successfully

◆ oriStringifyReturnStatus()

const char * oriStringifyReturnStatus ( const oriReturnStatus_t  status)

This function converts an oriReturnStatus_t enum into a more descriptive string.

Keep in mind that return statuses are not meant to be very descriptive, as they serve simply to give a vague idea of whether or not a function succeeded in executing.

An example of a described enum is ORION_RETURN_STATUS_OK = "function executed successfully".

Parameters
statusthe oriReturnStatus_t enumerator to stringify
Returns
a string that describes the status enumerator
