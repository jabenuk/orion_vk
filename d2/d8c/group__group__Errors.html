Detailed Description

Functionality found in this module is related to catching and debugging errors thrown by the library at runtime. You should also see the Debugging page for more information about debugging.

It is part of the core Orion library interface.

◆ oriReturnStatus

enum oriReturnStatus

All Orion functions (that could result in errors) return an oriReturnStatus enum. If the function returns any value other than ORION_RETURN_STATUS_OK (0), something has gone wrong. The name of the returned enum will give some information, but this is generally vague and you should check the debug output for more information.

This can be used to manage errors in your program.

◆ oriEnableDebugMessages()

void oriEnableDebugMessages ( oriErrorSeverityBit  severities)

A list of error severities can be seen in the description of oriErrorCallback.

The specified error callback (if none is given, then the default one) will be called when a debug message that matches the specified criteria is enqueued by the Orion library.

You can pass the ORION_ERROR_SEVERITY_ALL_BIT enum to the severities parameter to enable all debug messages.

Parameters
severitiesa bit field of severities to enable

◆ oriSetErrorCallback()

void oriSetErrorCallback ( oriErrorCallback  callback,
void *  pointer 
)

User data can be passed to the callback function (oriErrorCallback) through the pointer parameter. This will be recieved in the callback with the parameter of the same name.

Pass NULL to the callback parameter to use the default, built-in error callback.

Pass NULL to the pointer parameter if you do not want to pass any data to the callback.

Parameters
callbackNULL or the callback function to use.
pointerNULL or specified user data that will be sent to the callback.
