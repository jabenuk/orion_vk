The callback that will be used by this debug messenger is based on the Orion error callback; that is to say, a message will be generated by Orion with Vulkan-reported information, such as severity, message, type, etc, and will be reported to the Orion callback in the message parameter. The code of the error will be 0x03, and the error name will be VULKAN_DEBUG_MESSENGER.

The user pointer that will be reported to the callback is specified in oriSetErrorCallback(). You do not have to filter the messages yourself.

Note
The VK_EXT_debug_utils extension must be enabled; if not, ORION_RETURN_STATUS_EXT_NOT_ENABLED will be returned, and debugMessengerPtr will be set to NULL.
Parameters
statethe state object to which the resulting VkDebugUtilsMessengerEXT will be tied.
instancethe Vulkan instance that the messenger will be used with.
extequivalent to the pNext parameter in the Vulkan Specification (linked below): NULL or a pointer to a structure extending this structure.
debugMessengerPtra pointer to the structure to which the debug messenger will be returned.
severities(bitmask) severities of the messages to display.
types(bitmask) types of the messages to display.
Returns
the return status of the function. If it is not 0 (ORION_RETURN_STATUS_OK) then a problem occurred in the function, and you should check the debug output for more information.

void oriSpecifyInstanceDebugMessages ( oriState *  state,

)

This function defines part of the creation info of instance debug messengers, which - if ORION_FLAG_CREATE_INSTANCE_DEBUG_MESSENGERS is enabled and VK_EXT_debug_utils is specified - are automatically created in oriCreateInstance().

This function only has an effect if the VK_EXT_debug_utils extension has been specified. This doesn't necessarily have to be done before this function is called (as long as it is before instance creation) but doing so will generate a warning nonetheless. The ORION_FLAG_CREATE_INSTANCE_DEBUG_MESSENGERS flag must also be enabled - again, this doesn't need to be done before but a warning will be generated if such a scenario were to occur.

Note
The message filters specified in this function are different to those specified in oriEnableLibDebugMessages(). Whilst those specified in that function are related to the Orion library, those specified here are related to the Vulkan API.
Parameters
statethe state to modify.
severities(bitmask) severities of the messages to display. By default, no messages are displayed (even if ORION_FLAG_CREATE_INSTANCE_DEBUG_MESSENGERS is true)
types(bitmask) types of the messages to display. By default, no messages are displayed (even if ORION_FLAG_CREATE_INSTANCE_DEBUG_MESSENGERS is true)
