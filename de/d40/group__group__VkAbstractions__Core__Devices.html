Detailed Description

Functionality in this module is related to the management and usage of graphics devices with Vulkan, including the enumeration of available and suitable physical devices and choosing the best one for the application.

This module is part of the core Vulkan API module.

Typedef Documentation

◆ oriPhysicalDeviceSuitabilityCheckFunc

typedef bool(* oriPhysicalDeviceSuitabilityCheckFunc) (VkPhysicalDevice device)

A function of this signature is to be used to select suitable graphics device/s to be used in the application.

Devices reported to this function will only include those with Vulkan support. If there are no graphics devices with Vulkan support, this function will not be called.

You should return true if the device supports the extensions you require, the device supports the required queue families, etc. You can make the function as simple or as advanced as you need - just make sure you return true if the device is suitable and false if it isn't.

Parameters
devicethe device to be queried
Returns
true if the device is suitable to be used.
false if the device is not suitable to be used.

The resulting logical device will be managed by state.

If physicalDeviceCount is 1, then physicalDevices is a pointer to the desired physical device. If physicalDeviceCount is more than 1, then physicalDevices is an array of desired physical devices in the same device group. physicalDeviceCount cannot be 0.

See Vulkan Docs/VkDeviceCreateInfo and, occasionally, Vulkan Docs/VkDeviceGroupDeviceCreateInfo for more info.

Parameters
statethe state which will manage the resulting logical device.
physicalDeviceCountthe amount of physical devices to create the logical device for.
physicalDeviceseither a pointer to the physical device or an array of physical devices (see above).
extequivalent to the pNext parameter in the Vulkan Specification (linked below): NULL or a pointer to a structure extending this structure.
queueCreateInfoCountthe size of the queueCreateInfos array.
queueCreateInfosarray of VkDeviceQueueCreateInfo structures describing the queues that are requested along with the logical device.
extensionCountthe size of the extensionNames array.
extensionNamesarray of null-terminated UTF-8 strings containing the names of device extensions to be enabled for the logical device.
enabledFeaturesNULL or a pointer to a VkPhysicalDeviceFeatures structure containing flags of device features to be enabled.
deviceOutpointer to the variable into which the resulting device will be returned.
Returns
the return status of the function. If it is not 0 (ORION_RETURN_STATUS_OK) then a problem occurred in the function, and you should check the debug output for more information.

This function retrieves an array of physical devices accessible to a Vulkan instance that are considered suitable for the application.

Note
For a list of all available physical devices, pass NULL to the checkFunc parameter. This will mean that no suitability checks will be done, and all devices with Vulkan support will be returned.
Parameters
instanceVulkan instance to query
checkFuncan oriPhysicalDeviceSuitabilityCheckFunc function pointer that returns true if a device is suitable and false if not.
countOutpointer to a variable into which the number of suitable physical devices will be returned.
devicesOutpointer to an array into which the list of suitable physical devices will be returned. This array will be allocated by the function.
Returns
the return status of the function. If it is not 0 (ORION_RETURN_STATUS_OK) then a problem occurred in the function, and you should check the debug output for more information.
