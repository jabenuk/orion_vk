It is part of the core Orion library interface.

This function initialises the Orion library.

It also initialises a given VkInstance struct (or array of VkInstance structs, depending on the amount given in instanceCount.)

The parameters apiVersion, applicationName, applicationVersion, engineName, and engineVersion will be passed to Vulkan via the creation of a VkApplicationInfo structure.

If enabledLayerCount is not 0, then enabledLayers must be an array of null-terminated strings containing the names of each layer to enable for the instance(s) (if enabledLayerCount is 1, then you can simply pass a pointer to a string). This logic also applies to the enabledInstanceExtensionCount and enabledInstanceExtensions parameters.

Note
Specifying the same layer or extension multiple times will cause problems, as duplicates are not accounted for. It is on you to avoid duplicates.
Parameters
instanceCountthe amount of instances to create. This should be 1 in almost all cases, and cannot be 0.
instanceOutif instanceCount was 1, then this is a pointer to the VkInstance struct to initialise. Otherwise, it is an array of VkInstance structs.
instanceFlagsa bitmask of VkInstanceCreateFlagBits indicating the collective behaviour of the instances.
apiVersionthe Vulkan API version.
applicationNameNULL or the name of the application.
applicationVersionthe version of the application.
engineNameNULL or the name of the engine.
engineVersionthe version of the engine.
enabledLayerCountthe amount of layers to enable for the instances.
enabledLayersan array of the names of the layers to enable for the instances.
enabledInstanceExtensionCountthe amount of extensions to enable for the instances.
enabledInstanceExtensionsan array of the names of the extensions to enable for the instances.
instanceNextNULL or a pointer to a structure to extend the instance creation info structures.
Returns
OK if executed successfully
SKIPPED if the library was already initialised or if an instance was already created to instanceOut
NO_OUTPUT if instanceOut is NULL
NULL_POINTER if enabledLayers is NULL but enabledLayerCount is more than 0, or for the enabledInstanceExtension* equivalents
ERROR if there was an unspecified error, or if memory for the instance failed to allocate

◆ oriSetVulkanAllocators()

const oriReturnStatus_t oriSetVulkanAllocators ( VkAllocationCallbacks *  callbacks)

This optional function sets the internally-held structure in which allocation function pointers can be defined.

The structure passed to the callbacks parameter of this function will be referenced in any Vulkan function called internally by Orion with a pAllocator parameter.

Passing NULL to this function will reset Vulkan to using default allocation callbacks as described by the implementation.

You can retrieve these callbacks with oriGetVulkanAllocators().

Parameters
callbacksNULL or the callbacks structure to use with Vulkan functions.
Returns
OK if executed successfully

const VkAllocationCallbacks * oriGetVulkanAllocators ( )

This function gets the internally-held structure in which allocation function pointers are defined. This structure can be set with oriSetVulkanAllocators().

If no custom memory allocation functions have been set, this function will return NULL.
